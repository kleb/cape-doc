7.15. Write SPLITMQ input file using OVERFLOW namelist¶

Create a splitmq input file for extracting the L=1 surface from all walls using an OVERFLOW namelist as grid information about surface grids.

By changing the first two lines of the resulting file, an input file for splitmx can also be created.

Call
$ po_WriteSplitmq.py $NML $SPLITMQ [OPTIONS]
Inputs

  • NML: Name of OVERFLOW namelist file

  • SPLITMQ: Name of output splitmq input file

Options
-h, --help

Display this help message and exit

-i NML

Use NML as input namelist file {overflow.inp}

-o SPLITMQ

Create SPLITMQ as output {splitmq.i}

--wall

Only include walls (by default, thrust BCs are also included)

Versions

  • 2016-12-19 @ddalle: First version

Methods:

This script can also be imported as a module if the scriptlib/ folder is on the Python path. If so, the following methods are created.

po_WriteSplitmq.Nml2Splitmq(*a, **kw)¶

Convert a Tecplot PLT file to a Cart3D annotated triangulation (TRIQ)

Call
>>> Nml2Splitmq(fnml)
>>> Nml2Splitmq(fnml, fsplitmq)
>>> Nml2Splitmq(i=fnml, o=fsplitmq)
Inputs
fnml: {"overflow.inp"} | str

Name of OVERFLOW namelist input file

spitmq: {"splitmq.i"} | str

Name of splitmq input file

wall: True | {False}

Only include walls if True; otherwise include thrust BCs, too
